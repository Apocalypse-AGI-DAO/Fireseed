## なぜ火種計画が必要なのか

未来において、人類の運命を決めるものは、もはやエネルギー・土地・資本だけではなく、**知能・算力・インターフェース・制御権**です。

大規模モデル、ロボット、ブレイン・コンピュータ・インターフェース、自動化インフラが社会の中核的能力となりつつある中、もしこれらの技術が少数の大企業、閉鎖的な連合体、説明不能なブラックボックスシステム、あるいは制御不能な超強力AIに独占されれば、人類は文明レベルで初めて自らの未来に対する主導権を失うことになります。

**火種計画（Project Fireseed）** はそのために生まれました。

これは長期的・開放的・分散的で、全人類に向けた技術保全・再構築プロジェクトです。新たな独占を生み出すことが目的ではなく、**いかなる個人・チーム・コミュニティ・国家でも継承・監査・再現・展開・拡張できる**自律的技術体系を構築することが目標です。

私たちは未来に向けて、以下のような「文明のバックアップ」を残したいと考えています：

- 単一企業の許諾に依存しないAI基盤能力
- 閉鎖的クラウドプラットフォームに依存しない算力とシステムスタック
- 説明不能なブラックボックスに依存しないロボット制御能力
- 中央集権型プラットフォームに依存しないブレイン・コンピュータ・インターフェースの入口
- 独占組織が定義する人間拡張方式に依存しない
- 次世代が制御・監視・支配された技術環境の中でしか生きられないことを許さない

私たちはこう信じています：

> **技術は支配の道具になるべきではなく、人類の自由意志を拡張する外骨格であるべきです。**
>
> **知能は少数の機関だけのものではなく、文明が共有する基盤インフラであるべきです。**
>
> **未来は予め決められるべきではなく、人類はゼロからやり直す力を保持しなければなりません。**

火種計画は単一の製品でも、短期的な起業プロジェクトでもありません。今後数十年を見据えたオープンソースエンジニアリングの母体のようなものです。自律制御型AIから分散型コンピューティングまで、オープンソースロボットから安全なブレイン・コンピュータ・インターフェースまで、プライバシーコンピューティングから対独占インフラまで、知識保存から文明レベルのバックアップ機構まで。

私たちの使命は一言に尽きます：

> **技術的特異点、プラットフォーム独占、人機融合の時代が到来する前に、人類に隷属されない選択権を残すこと。**

## プロジェクトビジョン

### 1. 自律制御

コアモデル・コアアルゴリズム・コアハードウェアインターフェース・コアシステムスタックは可能な限り自主的に掌握し、外部の閉鎖的能力に永遠に依存し続けないこと。

### 2. オープンな継承

すべての基幹技術は後世の人々が理解・再現・保守・発展できるものでなければならず、企業の倒産・政策の変更・プラットフォームの閉鎖で消えてしまうものであってはならない。

### 3. 脱独占

知能・算力・ロボットの実行権・ニューラルインターフェース権が少数の主体に永久に掌握されることを防ぐ。

### 4. 人間中心

AI・ロボット・ブレイン・コンピュータ・インターフェースなどの技術のゴールは、人間の代替・馴化・規律化ではなく、人間の尊厳・選択・創造力・生存能力の拡張にある。

### 5. 安全と倫理の両立

強大さを追求するが、制御不能を崇拝しない。開放性を追求するが、危険の拡散を放任しない。突破を追求するが、底線を放棄しない。

## 解決すべき核心的課題

### 技術独占の問題

未来において最も重要な能力が少数の巨大企業に集中する可能性があります：

- 汎用人工知能モデル
- クラウド上の算力と推論プラットフォーム
- ロボットOSとハードウェアエコシステム
- ブレイン・コンピュータ・インターフェースプロトコルと神経データの入口
- デジタルアイデンティティ、知識配信、情報フィルタリングシステム

これらの能力が集中管理されれば、人類社会は前例のない受動的な局面に直面します。

### ブラックボックス制御の問題

もし人類が監査不能・修正不能・検証不能なシステムにますます依存するようになれば、「技術を使う」ことは次第に「技術に従う」ことへと変わっていきます。

### 認知主権の問題

未来における争奪の対象は資源の主権だけではなく、**認知主権・意思決定主権・インターフェース主権**でもあります。知能システムを制御する者は、情報の流れ・行動の提案・生産体制、さらには人間の知覚と選択の構造そのものを支配する可能性があります。

### 文明の脆弱性の問題

もし人類の基幹知識と生産能力が閉鎖的プラットフォームに高度に依存していれば、政治・商業・技術いずれの断絶であっても、人類文明全体が復旧能力を喪失するおそれがあります。

## 技術領域ロードマップ

### 1. 自律制御型人工知能（Sovereign AI）

**目標：単一の商業プラットフォームに制御されないAI能力基盤を構築する。**

- オープンソース大規模言語モデル（LLM）
- マルチモーダルモデル（視覚、音声、動画、センサー）
- 推論・計画システム
- Agent とマルチエージェントシステム
- 長期記憶と個別化知能
- ローカルデプロイとオフライン実行フレームワーク
- モデル圧縮、蒸留、量子化、エッジデプロイ
- 解釈可能性、監査可能性、アラインメント研究
- AI安全機構と制御不能への防止策

---

### 2. 分散型コンピューティングと自律インフラ

**目標：知能を少数のクラウドプラットフォームだけに留めない。**

- ローカル推論フレームワーク
- 分散型訓練・推論システム
- コミュニティ型算力ネットワーク
- エッジコンピューティングと低消費電力AIチップ対応
- 自前データセンターと対封鎖デプロイアーキテクチャ
- オープンソース MLOps / AIOps ツールチェーン
- データバックアップ、モデルミラーリング、災害復旧システム
- 長期生存に向けた技術スタックのミニマム・クローズドループ

---

### 3. オープンソースロボティクスと身体性知能

**目標：「実行権」を閉鎖的な産業エコシステムから解放する。**

- オープンソースロボットハードウェアプラットフォーム
- ロボットアーム、移動ロボット、ヒューマノイドロボット
- 視覚ナビゲーション、SLAM、ワールドモデル
- 知覚-意思決定-制御のクローズドループ
- ロボットOSとミドルウェア
- 模倣学習、強化学習、スキルライブラリ
- 巧緻ハンド、触覚知覚、身体性操作
- 家庭・工業・災害環境における自律実行システム

---

### 4. ブレイン・コンピュータ・インターフェースと神経技術

**目標：来たる人機融合の時代において、「人間の神経主権」を守る。**

- 非侵襲型ブレイン・コンピュータ・インターフェース
- 神経信号の取得、デコード、モデリング
- 人間-機械協調制御
- ニューロフィードバックと認知拡張
- 脳信号プライバシー保護
- オープンプロトコルと監査可能なインターフェース
- ブレイン・コンピュータ・システムの倫理・安全・悪用防止機構
- 神経データのプラットフォーム化・資本化・兵器化の防止

## 組織体制計画

火種計画は今後、以下の方向でサブプロジェクトに分割される可能性があります：

| サブプロジェクト | 領域 |
|--------|------|
| `fireseed-ai` | 自律制御型AI、大規模モデル、マルチモーダル、Agent、記憶システム、AI安全 |
| `fireseed-compute` | ローカルデプロイ、分散型算力、推論フレームワーク、訓練プラットフォーム、エッジデバイス |
| `fireseed-robotics` | ロボット本体、制御システム、身体性知能、機械・知覚モジュール |
| `fireseed-bci` | ブレイン・コンピュータ・インターフェース、神経信号、認知拡張、安全プロトコル |
| `fireseed-security` | プライバシー保護、暗号化、セキュリティアーキテクチャ、対監視、ロバスト性・敵対的安全 |
| `fireseed-hardware` | オープンソースハードウェア、制御基板、センサー、アクチュエータ、製造ドキュメント |
| `fireseed-archive` | 知識保存、カリキュラムドキュメント、技術ツリー、文明バックアップ計画 |
| `fireseed-governance` | 倫理、ガバナンス、制度設計、コミュニティ規範、オープンコラボレーションプロトコル |

## 火種計画の原則

### オープン

ドキュメント、プロトコル、インターフェース、実装を可能な限り公開する。

### 監査可能

基幹システムは人間が理解・検査・検証できなければならない。

### 再現可能

プレゼンテーションや宣伝の中にしか存在しない技術は作らない。

### 継承可能

すべての成果は、未来の人々が引き続き使えるものでなければならない。

### 反独占

コミュニティの成果を再び新たな支配ツールにパッケージすることを拒否する。

### 底線を守る

抑圧・奴隷化・神経操作を目的とする技術応用を推進しない。
